# BADM 350 — Spring 2026 Case Studies

Student team case studies from **BADM 350: Technology & AI Strategy**, Spring 2026, taught by Vishal Sachdev at the University of Illinois Urbana-Champaign (Gies College of Business).

Each case study documents an automation a student team built on their semester project using [Codex](https://chatgpt.com/codex) during Week 13 (Agentic Workflows & Pre-Launch Toolkit).

## How to read these

Each team's file lives in `case-studies/team-<name>.md`. Every file follows the same structure (see [`_TEMPLATE.md`](./_TEMPLATE.md)):

- **Product** — what the team's semester project is
- **Automation** — what they built with Codex
- **Prompt used** — the actual prompt or instructions given to Codex
- **Inner/outer loop annotation** — where AI execution ended and human judgment began (Wissner-Gross framing)
- **Anthropic pattern** — which of the five agent patterns from *Building Effective Agents* this automation matches
- **What required human judgment** — where the automation wasn't autonomous
- **What didn't work** — honest reflection on limitations

## For future BADM 350 students

These are real examples from your peers one year ahead of you. Read them before Week 13 to see what's possible. The instructor's picks for each semester (3–5 standouts) are also mirrored into [`codex-for-badm350`](https://github.com/vishalsachdev/codex-for-badm350/tree/main/case-studies).

## How to submit (students)

1. Fork this repo from your team account or one team member's account
2. Create a branch: `git checkout -b team-<name>-case-study`
3. Copy `_TEMPLATE.md` to `case-studies/team-<name>.md` and fill it in
4. Commit, push, open a PR to `vishalsachdev/badm350-spring-2026-case-studies:main`
5. Submit the PR URL to Canvas (Week 13 → Capstone assignment)
